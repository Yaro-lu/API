# 客户端导航与静态页面设计

日期：2026-07-14

## 目标

在不改变当前后台启动、工作流、任务、账号和隧道逻辑的前提下，把现有客户端扩展为可承载独立产品功能的桌面控制台。本阶段只完成信息架构、页面导航和静态界面，除页面切换外不接入新业务功能。

## 设计方向

保留现有浅蓝背景、白色卡片和蓝紫主色，采用克制、清晰的桌面控制台风格。左侧使用固定白色导航栏，右侧使用全局标题栏、页面内容区和状态栏。现有首页完整保留为“概览”，避免重写已经工作的界面与运行逻辑。

不采用整套重做方案，原因是当前首页已经经过使用验证；不采用多弹窗方案，原因是工作流、任务、网络和设置属于长期使用的一级模块，需要稳定入口和统一空间关系。

## 信息架构

1. 概览：保留当前状态灯、URL、API Key、工作流、任务进度和快捷操作。
2. 接口服务：展示文字、图片、视频兼容协议和模型路由规划。
3. 工作流：展示已发现的工作流包、英文服务标识、输出类型和依赖状态。
4. 任务中心：展示异步队列、任务状态、重启恢复语义和历史记录结构。
5. 网络访问：展示本机、局域网、公网三种访问模式及大文件传输提示。
6. 资源管理：展示客户端、运行环境、模型和工作流的分包关系。
7. 设置：展示安全、并发、存储和默认模型设置的未来入口。

## 交互边界

- 本阶段可用交互仅为左侧页面切换，以及原概览页已经存在的功能。
- 新页面中的按钮、开关和输入区域使用静态或禁用状态，并明确标记“待接入”。
- 页面切换不得重启后台服务，不得清空任务，也不得改变已有运行状态。
- 运行环境缺失页只替换概览内容，不覆盖左侧导航和其他静态页面。

## 布局与适配

- 默认窗口扩大到适合控制台的尺寸，保留最小窗口约束并继续居中显示。
- 左侧导航固定宽度；右侧页面内容自适应横向空间。
- 页面内容控制在单屏主要信息范围内；较长说明使用精简卡片，避免首版引入复杂滚动容器。
- 中文使用 Microsoft YaHei UI，代码、URL 和模型标识使用 Consolas。

## 验证标准

- 七个导航入口均可切换，标题与激活状态同步。
- 概览页原有引用控件仍存在，健康检查更新不会因切页报错。
- 新页面在默认尺寸和最小尺寸下不发生明显遮挡、溢出或重叠。
- Python 语法检查和现有自动化测试通过。
- 重启客户端后逐页截图检查；视觉无明显问题后暂停，交给用户确认。

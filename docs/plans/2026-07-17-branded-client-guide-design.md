# 品牌启动入口、项目教学与环境包下载引导设计

## 目标

让轻量客户端安装后更像正式 Windows 软件，并让首次使用者能够在没有自动环境部署的情况下完成后续操作。最终安装目录应直接出现带项目 Logo 的“灵境造片厂”启动入口和《灵境造片厂使用教学.pdf》；所有涉及在线安装或修复的按钮都必须明确引导用户前往 GitHub 手动下载环境包。

## 方案选择

启动入口采用 Inno Setup 在安装根目录创建 Windows 快捷方式，同时继续保留开始菜单和可选桌面快捷方式。这个方案直接复用已经验证过的 `runtime\python\pythonw.exe` 启动链和 `app.ico`，不增加新的二进制运行时或编译器依赖。Windows 资源管理器会显示项目 Logo 和“灵境造片厂”名称，用户无需接触 `start.bat`。

教学文档使用 ReportLab 生成稳定 PDF，源生成器和最终 PDF 一并保留在仓库中。发布脚本把 PDF 从 `docs/` 映射到安装根目录，构建时将其视为必需成员。文档覆盖安装、环境包、模型、工作流、URL/API Key、主要 HTTP 接口、异步任务流程和常见问题。PDF 需要使用中文字体、页眉页脚和页码，并经过 Poppler 渲染和逐页视觉检查。

环境入口不再尝试自动下载。`_install_runtime_from_mirror` 改为打开模态引导窗口，展示 `https://github.com/Yaro-lu/API`，提供“复制地址”“打开 GitHub”“选择本地环境包”三个动作。资源页、缺失环境页和高级维护弹窗统一改为“获取环境包”措辞，避免承诺不存在的一键部署。下载后仍使用现有本地 7z 安装与 SHA256 校验链，因此不会降低现有安装安全性。

## 验证

自动测试覆盖根目录品牌快捷方式、PDF staging 契约、按钮文案、弹窗地址和复制动作；完整测试套件验证回归。随后真实构建轻量安装包，在隔离目录安装，核对根目录快捷方式和 PDF，启动品牌入口，打开环境包引导弹窗并验证剪贴板，再卸载清理。

# 运行环境安装进度界面设计

## 目标

运行环境下载和安装过程中只展示用户能理解的阶段、模块与进度，不再把正常状态显示成红色或橙色警示。命令输出继续在后台处理，只有真实失败才进入错误状态。

## 界面

下载与安装统一使用简洁进度弹窗：标题、当前阶段、横向进度条、简短说明。正常阶段只使用 `text`、`text2`、`muted` 和品牌主色；不使用 `warn` 或 `error`。

下载进度来自实际接收字节，显示 0–100%。安装进度使用可验证的里程碑：

1. 准备安装
2. 校验安装包
3. 检查环境包结构
4. 安装核心模块（Python、ComfyUI、Torch/CUDA、网络组件）
5. 验证运行模块
6. 停止后台服务并切换环境
7. 重启客户端完成安装

进度条表示当前安装阶段，不伪造逐文件百分比。安装期间关闭窗口只更新中性提示，不弹出警告色文字。

## 错误处理

网络下载失败仍进入现有手动下载回退页。环境安装失败时，进度窗口明确显示“安装未完成”和简短原因；红色只允许出现在这个真实失败状态。详细解压器输出不在正常界面展示。

## 验收

- 下载和安装弹窗均包含 `ttk.Progressbar`。
- 正常状态标签不使用 `C["warn"]` 或 `C["error"]`。
- 下载能按实际字节更新百分比。
- 安装流程依次报告模块化里程碑。
- 现有 SHA256、目录安全校验、事务更新和失败回滚逻辑不变。

# 灵镜造片厂客户端最终审计与发布计划

> 日期：2026-07-16
> 基线：`6f0b72956d3fbbc52a7629f896209c43ba3bf388`
> 回滚标签：`checkpoint-pre-final-audit-20260716`

## 目标

在不移动或删除现有模型、用户输出的前提下，将客户端整理为可审计、可回滚、可在全新 Windows 10 上安装运行的版本。正式发布物不得包含账号令牌、API Key、日志、请求记录、用户作品、模型或 Git 元数据。

## 已确认的发布阻断

1. 旧 `app/worker_service` 不再被正式入口使用，但保留了无鉴权管理接口、路径穿越和 SSRF 风险。
2. 旧便携包与安装器会误收模型、运行状态和用户数据，且缺少实际启动所需文件。
3. `/health` 公网免鉴权返回完整任务和运行信息；生成 Key 同时可以调用管理接口。
4. 请求大小、生成参数、任务记录和请求文件没有完整上限；并发提交存在任务槽竞态。
5. API Key、平台会话令牌和启动日志存在明文泄露风险；远程平台地址允许非本机 HTTP。
6. 环境安装允许缺少校验文件并直接覆盖正式环境，缺少隔离验证和失败回滚。
7. README、版本号、运行入口和打包脚本互相不一致。
8. 当前机器没有代码签名证书；因此只能降低误报概率，不能承诺所有杀软和 SmartScreen 永不提示。

## 实施顺序

### 1. 基线与影响分析

- 保留已推送的基线提交与标签。
- 每个共享符号修改前执行 GitNexus impact；高风险改动先记录直接调用方和回归范围。
- 删除前用代码搜索、AST 可达性和测试确认孤岛模块。

### 2. 删除旧实现与脏内容

- 删除 `app/worker_service` 及其专属配置、旧启动脚本和旧手工测试。
- 删除不可达的旧任务队列/存储模块。
- 删除失效的 AIWorker 安装器与旧便携构建逻辑，用新的白名单发布链替代。
- 重写 README，仅保留真实入口、真实目录结构和用户操作。
- 保留 `models/`、`outputs/`、当前工作流和运行环境；不清理用户内容。

### 3. 主 API 安全与稳定性

- 公开健康检查只返回状态和版本；完整状态迁移到鉴权接口。
- GUI 本地轮询携带 API Key。
- 生成 Key 与本机管理 Key 分离；Tunnel/工作流管理仅接受管理 Key。
- 使用常量时间比较；关闭 CORS credentials，只开放必要方法和请求头。
- 增加请求体、prompt、尺寸、steps 和频率上限。
- 在锁内原子预留任务槽；ComfyUI 入队失败时释放，失败请求不落完整正文。
- 任务记录和请求元数据设保留上限；不自动删除用户输出。
- 输出下载必须同时匹配真实任务和任务记录中的文件。
- 远端错误返回稳定错误码，不向公网暴露内部路径和异常细节。

### 4. 凭据与远程连接

- Windows 下使用 DPAPI 保护本地 API Key、管理 Key 和平台会话 Token，并兼容迁移旧明文字段。
- 日志不打印完整 Key；构建时扫描 `sk-local-`、Authorization、Session Token 等敏感模式。
- 正式平台只允许 HTTPS；开发例外仅限 `127.0.0.1`、`localhost` 和 `::1`。

### 5. 环境与模型供应链

- 固定当前官方环境包 SHA256；所有安装路径缺少可信校验即拒绝。
- 环境包先解压到 staging，检查路径、链接/reparse point、文件数和解压总大小，再事务替换；失败恢复旧环境。
- 模型导入/下载继续保留用户选择，但补充哈希清单入口和危险格式警告；不擅自删除现有模型。
- 将 CUDA 13 最低驱动要求修正为 580 系列，并在界面/文档中给出明确提示。

### 6. 启动、版本和发布链

- 统一版本为 `0.2.0`，正式入口使用 `runtime/python/pythonw.exe`，禁用用户级 site-packages。
- Trae 项目解释器固定为 `runtime/python/python.exe`，不再调用损坏的 `.venv/Scripts/python.exe`。
- 新安装器使用白名单文件清单，安装到当前用户的 LocalAppData，不要求管理员权限。
- 离线安装器内置客户端和干净运行环境，排除模型及所有用户数据，解决全新系统无 Python 时无法打开安装界面的死锁。
- 构建后生成成员清单、SHA256、运行时清单和依赖审计报告。

### 7. 验证矩阵

- Python 编译、`pip check`、依赖漏洞扫描、完整单元测试。
- 未鉴权健康检查、鉴权状态、管理权限、请求上限、并发预留、输出归属、HTTPS 限制、密钥迁移等回归测试。
- `PYTHONNOUSERSITE=1` 冷启动，确认不依赖系统 Python、7-Zip 或用户级包。
- 启动 GUI，检查后台状态、工作流、模型加载态、任务完成进度和彻底退出。
- 安装器静默安装到隔离目录后启动、调用 API、退出和卸载。
- 对发布物运行 Defender 扫描、敏感信息扫描、签名状态和 SHA256 检查。

## 发布边界

- 本轮交付：完整离线安装包（不含模型）和 SHA256；如体积允许，再提供完整便携包。
- 仓库保持私有时，不依赖匿名 GitHub Release 下载环境。
- 没有可信 Authenticode 证书时，发布物会保持透明、无 UPX、无 onefile 自解压、无混淆，但不能保证 SmartScreen 首次运行无提示。正式公开前应接入 Azure Artifact Signing/可信代码签名或 Microsoft Store。
- 开源许可证由项目所有者在公开前选择，本轮不擅自替项目指定许可证。

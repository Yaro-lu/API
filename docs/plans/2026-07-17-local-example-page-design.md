# 灵境造片厂本地示例页设计

## 目标

随轻量客户端安装一份完全本地、纯 HTML/CSS/JavaScript 的示例调用页。新手只需要从客户端复制 URL 和 API Key，即可连接客户端、自动发现可用模型，并按文字、图片、视频三类发起调用。页面不要求客户端新增接口，也不依赖线上网页、CDN、字体或脚本。

## 交互结构

- 顶部使用灵境造片厂 Logo 和品牌名称，明确标注“本地 API 示例页”。
- 首屏连接卡只要求填写“客户端 URL”和“API Key”，点击“连接并检测模型”。
- 连接成功后调用 `GET /v1/models`，按响应中的 `group`、`type` 或分组字段归类文字、图片、视频模型。
- 不可用模型仍显示，但标注“未就绪”且不可选择；每类自动选中第一个可用模型。
- 创作区使用三个清晰分类标签。图片为默认分类，因为它是新手最直观的验证方式。
- 结果区统一显示请求状态、耗时、错误信息和文字/图片/视频结果。

## 数据流

1. 用户输入基础 URL 和 Key。页面仅在当前浏览器会话保存 Key，基础 URL 可保存在本地浏览器。
2. 页面规范化 URL，兼容用户粘贴基础地址或已带常见接口后缀的地址。
3. 页面请求 `/v1/models`，解析 `models`、`data` 或三类数组。
4. 文字调用 `POST /v1/chat/completions`；图片调用 `POST /api/v3/images/generations` 并优先使用异步响应；视频调用 `POST /v1/workflows/run/{workflow_id}`。
5. 异步任务轮询 `GET /v1/tasks/{task_id}`。受保护的媒体文件通过带 Bearer Key 的 `fetch` 下载为 Blob 后展示。

## 安装交付

- 源文件：`examples/灵境造片厂示例页.html`。
- 发布构建将它复制为安装根目录 `灵境造片厂示例页.html`。
- 安装器创建桌面快捷方式“灵境造片厂示例页”，目标为根目录 HTML，并使用客户端同一 `app.ico`。
- 桌面图标任务默认选中，使正式客户端与示例页同时出现在桌面；用户仍可在安装器中取消。

## 错误处理与安全

- 401/403 分别提示 Key 错误或权限不足；无法访问时提示检查客户端是否启动以及 URL 是否正确。
- 没有可用图片模型时明确提示先在客户端完成环境和图片模型安装，不制造“已经可以生成”的假象。
- API Key 不写入 HTML、不写入日志、不进入长期本地存储。
- 页面中的动态文字使用 `textContent` 或转义后渲染，避免把服务端内容当 HTML 执行。

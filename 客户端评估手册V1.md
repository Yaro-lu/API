## 二、API 接口

### 基础信息
- **本地地址**：`http://127.0.0.1:18188`
- **公网地址**：由 Cloudflare Quick Tunnel 动态分配（如 `https://xxx.trycloudflare.com`）
- **鉴权方式**：Bearer Token（Header: `Authorization: Bearer sk-local-xxx`）
- **API Key**：启动时自动生成，写入 `runtime/session.json`，格式 `sk-local-{64位随机}`

### 端点

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| GET | `/health` | 否 | 健康检查（含任务进度、红绿灯状态） |
| GET | `/v1/workflows/list` | 是 | 已加载工作流列表 |
| POST | `/v1/workflows/run` | 是 | 执行默认工作流 |
| POST | `/v1/workflows/run/{id}` | 是 | 执行指定工作流 |
| GET | `/v1/tasks/status` | 是 | 当前任务执行进度 |

### 请求示例

```bash
# 执行默认工作流（Flux.2 文生图）
curl -X POST https://xxx.trycloudflare.com/v1/workflows/run \
  -H "Authorization: Bearer sk-local-xxx" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "a cute cat", "seed": 42, "steps": 20, "width": 1024, "height": 1024}'

# 查看进度
curl https://xxx.trycloudflare.com/v1/tasks/status \
  -H "Authorization: Bearer sk-local-xxx"
```

### 响应示例

**提交任务（异步）**：
```json

```

**健康检查（含进度）**：

## 四、现存问题

| 问题 | 严重度 | 说明 |
|------|--------|------|
| cloudflared 无状态重启 | 中 | cloudflared 崩溃后不会自动重连，需重启程序 |
| 无任务队列 | 中 | 同时只能执行一个任务，并发请求返回 429 |
| 无结果持久化查询 | 中 | 任务完成后输出文件无公开访问 URL（仅存在于 outputs/ 目录） |
| 工作流参数未泛化 | 中 | 参数注入按节点 `class_type` 硬编码匹配，新增工作流需扩展 server.py |
| `nodes_glsl.py` 缺少依赖 | 低 | ComfyUI 启动时 warn "Missing module `comfy_angle`"（已装但路径映射未覆盖） |

## 六、运行方式

### 一步启动

```batch
双击根目录 start.bat
```

GUI 启动后自动完成：

1. 启动 ComfyUI（端口 8188，highvram 模式，输出 → `outputs/`）
2. 启动 API 服务器（端口 18188，生成 API Key）
3. 启动 Cloudflare Quick Tunnel（公网 URL）
4. 红绿灯全部变绿 + 标题栏显示"就绪 ✓"后即可调用

### 通过服务器 API 调用示例

```bash
# 先查询健康状态（获取公网 URL 和 API Key）
curl http://127.0.0.1:18188/health

# 使用公网 URL 执行 Flux.2 文生图
curl -X POST https://xxx.trycloudflare.com/v1/workflows/run \
  -H "Authorization: Bearer sk-local-xxx" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "a cute cat", "seed": 42, "steps": 20, "width": 1024, "height": 1024}'
```
